# Pre-Depth を用いた深度推定精度向上の計画

作成日: 2026-07-06

## 背景

- ベースとする研究アイデア: 画像内の物体を切り出し、その領域だけで深度推定を行い、
  その結果 (pre-depth) を追加情報としてモデルに与えることで精度を向上させる。
- ベースモデル: Lotus-D (拡散モデルベースの単眼深度推定)。
- これまでの試み: YOLOv8-seg で生成したバイナリマスク (0/1) を
  4チャンネル Early Fusion (VAE 入力 conv 改造) で注入 → 精度向上せず。

## 前回うまくいかなかったと考えられる理由

1. **バイナリマスクの情報量不足**
   「そこに何かある」という objectness しか伝わらない。物体輪郭は RGB から
   VAE が既に抽出できるため、モデルにとってほぼ冗長。
   本来の研究アイデアは「物体領域の深度値そのもの (pre-depth)」を渡すことが肝。
2. **Early Fusion が事前学習を壊すリスク**
   VAE 入力 conv を 4ch に改造すると追加チャンネル分の重みが新規初期化になり、
   少ないデータ・短いステップでの微調整では SD 由来の表現を劣化させうる。
3. **マスク品質**
   YOLOv8n (最小モデル、COCO 実写で学習) は Hypersim の合成室内シーンでは
   検出漏れ・誤検出が多い。
4. **評価指標の問題**
   全体平均の abs_rel は面積の大きい床・壁に支配されるため、
   物体レベルの改善が数字に出にくい。

## この方式の急所: スケール・シフトの不整合

Lotus のような相対深度モデルは出力スケールが画像ごとに任意。
物体クロップごとに推定すると、各 pre-depth が「バラバラのものさし」になる。
これを揃えずに合成した pre-depth はグローバルに矛盾し、追加情報どころか
ノイズになる。**精度が上がらない場合、まずここを疑う。**

## 提案パイプライン

### Step 1: グローバル + ローカルの2段推定と整合合成(学習不要)

1. 全体画像で1回推定 → グローバル深度(粗いが整合的)
2. 物体クロップで個別推定
   - bbox に 2〜3割のコンテキスト余白を付ける
     (余白なしのクロップは深度モデルが文脈を失い精度が落ちる)
3. 各クロップの深度を、その領域のグローバル深度に
   **最小二乗でスケール・シフト合わせ**してから貼り込む

→ 「グローバルに整合していて、物体部分だけ高精細」な pre-depth が得られる。
BoostingDepth / PatchFusion と同じ原理で、**この合成だけで再学習なしに
境界精度が上がる**ことが知られている。まずこれをベースラインにする。

### Step 2: pre-depth の注入方法(潜在空間連結 + ゼロ初期化)

- Early Fusion (VAE 改造) はやめる。
- pre-depth を VAE でエンコードし、**潜在空間で連結**する:
  - pre-depth (1ch) を Lotus の深度処理と同様に「3ch 複製 + [-1,1] 正規化」で VAE に通す
  - UNet の conv_in を 8ch → 12ch に拡張し、**追加分をゼロ初期化**
  - ゼロ初期化により学習開始時点で公式 Lotus-D と同一性能が保証される
- pre-depth が存在する画素を示す**有効マスク**も併せて渡す。

### Step 3: 学習時の pre-depth は GT から合成

学習中に毎バッチ推論するのは遅すぎるため、GT 深度から擬似 pre-depth を合成する:

- 物体領域の GT 深度を切り出し、スケール・シフトの乱れ、ぼかし、欠損などの
  ノイズを加えて「推論で得られる pre-depth っぽく」劣化させる
- **条件ドロップアウト**(確率 ~30% で pre-depth をゼロに)
  → pre-depth への過依存を防ぎ、無い場合でも動作する
- ノイズ強度をランダム化 → 推論時の pre-depth 品質のばらつきに頑健になる

### Step 4: 損失と評価の改善

- 通常の損失に**マルチスケール勾配マッチング損失**を追加
  (pre-depth の恩恵は物体境界のシャープさに出るため、それを直接報酬にする)
- 評価は全体 abs_rel に加えて:
  - 物体境界の F 値 (boundary F-score)
  - 物体領域内の abs_rel(領域別評価)

### Step 5: 学習の出発点

- SD1.5 からではなく **公式 `jingheya/lotus-depth-d-v2-0-disparity` の重み**から
  微調整を開始する。pre-depth の使い方だけを学べばよくなるため、
  少ないデータ・ステップで済む。

## 進め方(ロードマップ)

1. **Step 1 の合成のみ実装して評価**(学習不要、短期間で結論が出る)
   - 上がれば方式の正しさを確認でき、上がり幅が
     「pre-depth の持つ情報の上限」の目安になる
2. **Step 2〜5 の学習込みパイプライン**で、モデル自身に融合を学ばせて
   Step 1 の上限を超えにいく

## 補足: その他の改善オプション(必要に応じて)

- Hypersim の GT セマンティックラベル (NYU40, `frame.IIII.semantic.hdf5`) を
  使ったオラクル実験。オリジナル Hypersim から `contrib/99991` の
  部分ダウンローダーで semantic のみ取得可能(数 GB 以下)。
  ※ 現在使用中の `omrastogi/Hypersim-Processed` には rgb と depth しか含まれない
- クロップのマスク生成を YOLO → SAM 等の強いモデルに置き換える
- fp16 → bf16 への変更(loss NaN 表示対策)、EMA 有効化、データ・ステップ増
- テスト時アンサンブル(flip / multi-crop 平均)、高解像度パッチ推論
